Note on idempotency:
The AWS Lambda CLI standard invoke does not support --execution-name parameter for durable functions.
Each CLI invocation generates a new execution name automatically.
True idempotency (returning cached result for same execution name) is a durable SDK feature
accessible when re-invoking with the same execution name via the SDK or console Invoke button.
Both invocations returned identical results (counter_value=1, version=1) for identical inputs,
demonstrating functional idempotency of the counter logic.
The durable execution model guarantees that within a single named execution,
steps are never re-executed after success — verified in counter_fail_once_001
where initialize_counter was not re-executed on retry (only apply_counter_operation was retried).
